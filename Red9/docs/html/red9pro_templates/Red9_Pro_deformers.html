original_geo_update_with_new_shape(original, newgeo, sync_uvsets=True)¶

<<< USE WITH CAUTION! >>>

replace the originalGeo cache in the “current” deformation stack, with the “newgeo” mesh shape. This has the effect of completely replacing the original mesh shape with the new shape whilst retaining all of the deformation stack. This is useful when clients send you updated meshes that need to replace master assets already rigged.

This transfers:
  • vertex positions
  • edge and face data
  • UVsets (we sync both uvset channels and the vtx data)
  • normals

Note

if the target mesh has different vertex indexes / counts, then the deformation of the source setup will be broken after running this and will need correcting.

Parameters:
  • original – the input mesh we’re going to update the target shape with
  • newgeo – the mesh who’s deformation stack we’re looking to update
  • sync_uvsets – True by default as we update the pointCache and that updates the UV data BUT the actually set membership doesn’t get updated by that process so we also need to create/ sync any missing uv sets

delete_softmod(deformer)¶

why, well if we’ve transferred a softMod from one mesh to another then the connections of all of the driver data, including the handle, will also be cast. With the softModHandle wired into multiple softMods it means that if you delete one of those softMods, both will get deleted in Maya, when the softMod code cleans up nodes.

This first checks if the softModHandle is wired to multiple softMod deformer nodes, and if so, it disconnect the handle from this deformer before then calling the standard delete

prune_blend_shape_selected_components(percentage=None, all_vtx=False, auto=False)¶

This acts on ALL targets in the blendshape Prune selected vertices from having any effect on ANY targets for this blendshape. The selection can be vertex, faces, edges, etc. Basically we remove those selected components from each target delta, and therefore from having any effect in the blendshape

If percentage and softSelect is on then we prune selected vertices with percent relative to the softSelect If percentage and softSelect is off then we prune selected vertices with percent on selected verts only If not percentage we prune the selected verts from the target shape completely so they have no effect. This doesn’t then consider softSelect at all

Parameters:
  • percentage – if not None (default) then we mix down the deltas for the given vertices by this percentage, with 100% having no effect and 0% removing the delta all together
  • all_vtx – if True we prune / process ALL vtx in the given object
  • auto – if True then we automatically switch the all_vtx flag when we don’t have any vertices actually selected. This is primarily for UI integration
prune_target_selected_components(target, percentage=None, all_vtx=False, auto=False)¶

This acts only on the given target in this blendshape Prune selected vertices from having any effect on a specific target of this blendshape. The selection can be vertex, faces, edges, etc.

If percentage and softSelect is on then we prune selected vertices with percent relative to the softSelect. If percentage and softSelect is off then we prune selected vertices with percent on selected verts only. If not percentage we prune the selected verts from the target shape completely so they have no effect. This doesn’t then consider softSelect at all

Parameters:
  • target – int or string, index value of the target or name of the target
  • percentage – if not None (default) then we mix down the deltas for the given vertices by this percentage, with 100% having no effect and 0% removing the delta all together
  • all_vtx – if True we prune / process ALL vtx in the given object
  • auto – if True then we automatically switch the all_vtx flag when we don’t have any vertices actually selected. This is primarily for UI integration
 
>>> # to remove the selected vertices from the target 'mytarget01' on 'blendShape1'
>>> blend = r9deformer.BlendShape('blendShape1')
>>> blend.prune_target_selected_components('mytarget01', all_vtx=False)
>>> 
>>> # to mix down just the selected vertices effect on the given target to 85% of their current
>>> blend = r9deformer.BlendShape('blendShape1')
>>> blend.prune_target_selected_components('mytarget01', percentage=85, all_vtx=False)
>>> 
>>> # to mix down the full targets effect to 85% of it's current
>>> blend = r9deformer.BlendShape('blendShape1')
>>> blend.prune_target_selected_components('mytarget01', percentage=85, all_vtx=True)
prune_target_components(target, vertices=[], vtx_percentages={}, percentage=None)¶

Prune given vertex deltas for a given target in the blendshape. This is wrapped by the other prune functions. If percentage is given then we use the vtx_percentages arg, else we use the vertices list.

If percentage and softSelect then we prune with percent relative to the softSelect (using vtx_percentages arg: found from the self.get_selected_vertices_soft()). If percentage and not softSelect we prune with percent on selected verts only (using vertices arg: found from the self.get_selected_vertices()). If not percentage we prune the given verts from the target shape completely so they have no effect. This doesn’t then consider softSelect at all.

Parameters:
  • target – int or string, index value of the target or name of the target
  • vertices – [] list of strings of vertex ids on full including mesh name eg: [‘geo.vtx[1]’, ‘geo.vtx[21]’, ‘geo.vtx[10]’]
  • vtx_percentages – {} dict of soft selection data where the key is the vtx and the value is the percentage of weight (0-1.0) eg: {‘pSphere.vtx[1]’:0.1, ‘pSphere.vtx[2]’:1.0, ‘pSphere.vtx[2]’:0.1}. Only valid if percentage is passed in
  • percentage – if not None (default) then we mix down the deltas for the given vertices by this percentage, with 100% having no effect and 0% removing the delta all together
