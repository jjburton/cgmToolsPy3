Pro_Pack : puppet : finger_pose_creator¶

ProPack puppet call to push a library of finger poses over to an existing PuppetRig system as finger sliders. The poses must be made using the “FINGERMIRRORED” pose handler in the Red9 PoseUI. There’s currently no UI support for this so please read carefully

>>> # import statement for the module via the r9pro decompiler
>>> from Red9.pro_pack import r9pro
>>> r9puppet_fingerposes = r9pro.r9import('r9puppet_fingerposes')

Specific Module to manage the creation of embedded finger poses on slider attrs for a given finger system.

This expects the finger mSystems to be properly setup as per the Red9 Puppet (see demo asset) and is used in our publishing systems. It’s exposed here to allow clients to update the internal poses we bake down to client PuppetRigs

class CreateFingerPoses(filepath, mSystem=None, handposes=[], fingerposes=[], chnrange=10, searchreplace=[], build_finger_ctrl=True)¶

Bases: object

This will add a new joint above each of the finger joints named SDK_grp which will take all the loaded pose information and bind that to the specified driver controller as a 0-100% channel. The poses given are r9Poses generated in a folder tagged with the FINGERMIRRORID handler.

Parameters:
  • filepath – filepath for the finger library
  • mSystem – the finger / toe mSystem nodemNode we’re processing, this gets bound to self.fingerSys which can be set after the class instance is taken but before the apply call if needed
  • handposes – FULL HAND poses : specific list of FULL HAND poses to apply in order
  • fingerposes – SPLIT FINGER poses : to be split and bound to EACH FINGER, this is a set of filter strings to run against the filepath
  • chnrange – range of the pose attrs added
  • searchreplace – if given this is used as a searchreplace call when generating the poseAttr from the pose file names allowing you to strip off specific parts of the name, ie ‘Mark_spread’ becomes an attr ‘spread’ with arg [‘Mark, ‘’]

Note

if the poses being loaded have transform data then that will be invalid as the new structure removes the initial transform values from each joint, leaving them with zero transforms at bind pose

>>> # import the module
>>> from Red9.pro_pack import r9pro
>>> r9puppet_fingerposes = r9pro.r9import('r9puppet_fingerposes')
>>> 
>>> # setup the variables
>>> filepath ='Q:/Demos/Poses/Red_Fingers'    
>>> handposes = ['bend_max', 'bend_min', 'spread_max', 'spread_min']
>>> fingerposes = ['bend_max', 'bend_min', 'spread_max', 'spread_min']
>>> 
>>> # create, or update a finger pose ctrl using the poses defined above
>>> mrig = r9Meta.getMetaRigs()[0]
>>> 
>>> # build left fingers
>>> creator = r9puppet_fingerposes.CreateFingerPoses(filepath, mSystem=mrig.L_ArmSystem.L_FingerSystem, handposes=handposes, fingerposes=fingerposes, chnrange=10)
>>> creator.apply_to_system()
>>> 
>>> # build right fingers
>>> creator = r9puppet_fingerposes.CreateFingerPoses(filepath, mSystem=mrig.R_ArmSystem.R_FingerSystem, handposes=handposes, fingerposes=fingerposes, chnrange=10)
>>> creator.apply_to_system()
